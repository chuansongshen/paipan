## 一、复核结论

### 1. 当前未发现已确认的严重问题

- 位置：`bazi.js:234-240`
- 结论：`tianDeMap['子']` 当前值为 `'巳'` 与常见天德贵人月支查法一致，不应改成 `'壬'`

复核后的常见天德表（月支 → 天德）：

| 月支 | 子 | 丑 | 寅 | 卯 | 辰 | 巳 | 午 | 未 | 申 | 酉 | 戌 | 亥 |
|------|----|----|----|----|----|----|----|----|----|----|----|----|
| 天德 | 巳 | 庚 | 丁 | 申 | 壬 | 辛 | 亥 | 甲 | 癸 | 寅 | 丙 | 乙 |

当前代码：

```js
const tianDeMap = {
  '子': '巳',
  '丑': '庚', '寅': '丁', '卯': '申',
  '辰': '壬', '巳': '辛', '午': '亥', '未': '甲',
  '申': '癸', '酉': '寅', '戌': '丙', '亥': '乙'
};
```

说明：

- 若把 `子` 改成 `壬`，会把“子月见巳”的常规命中改成漏判
- 同时会把“子月见壬”误判成天德贵人
- 这条原审查结论大概率是把其他神煞表或异表口径混入了天德表

## 二、可改进但暂不构成功能错误的问题

### 2. 天乙贵人采用日干与年干并行口径

- 位置：`bazi.js:212-220`
- 结论：代码同时以日干和年干查天乙贵人，属于兼容常见在线排盘站的宽口径实现，不宜直接定性为错误。当前实现已用 `天乙贵人(年)` 区分年干来源，语义上基本完整。

当前代码：

```js
if (tianYiMap[dayGan]?.includes(zhi)) shenSha.push('天乙贵人');
if (tianYiMap[yearGan]?.includes(zhi)) shenSha.push('天乙贵人(年)');
```

建议：
- 若后续希望进一步统一神煞口径，可将“按年干命中”统一放到显式来源标签中展示
- 在没有明确产品口径前，不建议删除年干分支

### 3. 文昌贵人存在来源标注不清

- 位置：`bazi.js:326-331`
- 结论：当前代码同时支持日干与年干查法，这本身不一定错误，但命中后统一标注为 `文昌贵人`，会让来源不够清楚。

当前代码：

```js
if (wenChangMap[dayGan] === zhi || wenChangMap[yearGan] === zhi) shenSha.push('文昌贵人');
```

建议：
- 若保留当前宽口径，建议把年干命中改成 `文昌贵人(年)` 以保持和天乙贵人一致
- 在产品未明确要求“只按日干”前，不建议直接删除年干分支

## 三、低优先级问题

### 4. 十神判断逻辑不严谨

- 位置：`bazi.js:104-116`
- 问题：`sameElement` 分支未校验阴阳即直接返回 `劫财`。严格逻辑应为：同五行同阴阳 = 比肩，同五行异阴阳 = 劫财。当前代码先用 `dayStem === otherStem` 拦截了比肩，剩余同五行情况确实只剩异阴阳（因天干唯一性），所以**功能上不会出错**，但逻辑路径不清晰。

当前代码：

```js
if (dayStem === otherStem) return '比肩';
const sameElement = Math.floor(dayGanIdx / 2) === Math.floor(otherGanIdx / 2);
if (sameElement) return '劫财'; // 缺少阴阳校验
```

建议补充阴阳判断以提高可读性：

```js
if (sameElement) {
  return dayYang === otherYang ? '比肩' : '劫财';
}
```

### 5. 太极贵人重复推入同名

- 位置：`bazi.js:229-231`
- 问题：日干和年干分别命中时，均推入 `太极贵人`（同名），虽然最终有 `new Set` 去重，但语义不明确——无法区分是日干命中还是年干命中。

当前代码：

```js
if (taiJiMap[dayGan]?.includes(zhi)) shenSha.push('太极贵人');
if (taiJiMap[yearGan]?.includes(zhi)) shenSha.push('太极贵人');
```

建议：与天乙贵人保持一致，要么只用日干，要么区分标注。

### 6. 福星贵人、国印、金舆同样混用日干与年干

- 位置：`bazi.js:264-278`、`bazi.js:417-423`
- 问题：与天乙贵人、文昌贵人相同的混用模式。建议统一查法基准。
