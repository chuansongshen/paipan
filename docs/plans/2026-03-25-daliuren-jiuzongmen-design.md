# 大六壬九宗门三传重构设计

## 目标

1. 按九宗门条文重写 `src/utils/daliuren.js` 的三传判课顺序与各门取传规则。
2. 保持现有对外返回结构与页面展示不变，不修改 `直指` 原文与相关展示组件。
3. 在实现完成后，与 `https://www.yypan.cn/liuren/` 的排盘结果做批量比对，记录仍存在差异的时间点。

## 当前问题

- 现有 `getSanChuan` 逻辑混合了参考仓库补丁、简化涉害评分和若干专门分支，分派顺序已偏离九宗门条文。
- 已补上的 `别责`、`八专` 只是局部修正，不能保证整体判课顺序正确。
- `比用` 缺少独立分派，`涉害` 是工程化近似，`遥克` 的蒿矢与弹射先后顺序相反。
- `返吟` 缺少专门实现，当前仅依赖通用递传，无法满足“无依 / 无亲”条文。
- 页面虽然能在 `直指` 文本中看到“课体”字样，但那是静态原文，不是程序结构化输出；本次不改这一部分。

## 设计原则

1. 只重构三传算法，不扩大到天将、神煞、直指、页面组件。
2. 尽量复用现有四课、天盘、遁干等稳定逻辑，避免连带回归面过大。
3. 把“判门”和“取传”拆开，减少 `getSanChuan` 内部的硬编码分支交叉。
4. 保留必要日志和异常，确保回归失败时能明确定位到宗门分派阶段。

## 九宗门分派设计

### 1. 伏吟

- 条件：天盘与地盘同位。
- 规则：
  - 有贼克，以上神发用。
  - 无贼克，阳日取干上神，阴日取支上神。
  - 发用若自刑，中传改取另一端上神；中传再自刑，末传取冲。
  - 非自刑则按刑取中末。

### 2. 返吟

- 条件：天地盘相冲。
- 规则：
  - 有贼克，入 `无依`，仍以贼克/比用/涉害定初传，但中末改为相冲链。
  - 无贼克，入 `无亲`，取日支驿马为初传，中传取支上神，末传取干上神。

### 3. 常规局

按以下顺序判：

1. `贼克`
2. `比用`
3. `涉害`
4. `遥克`
5. `昴星 / 别责 / 八专`

#### 贼克

- 先取下贼上。
- 若无下贼上，再取上克下。
- 单一候选则直接发用。

#### 比用

- 多贼克或多上克时，先按与日干同阴阳筛选。
- 若只剩一个，直接发用。
- 若仍然多个，或一个都没有，再进入涉害。

#### 涉害

- 不再使用现有“12 支循环五行冲克计数”近似。
- 需要按候选课对应的临地盘线路统计所涉克害深浅。
- 若深浅不同，深者发用。
- 若深浅相同，再按孟、仲、季与“干上 / 支上”条文收束。

#### 遥克

- 条件：四课无贼克。
- 先判 `蒿矢`：上神遥克日。
- 无蒿矢再判 `弹射`：日干遥克上神。
- 多候选时同样先比用，再必要时进入涉害式收束。

#### 昴星 / 别责 / 八专

- 无遥无克且四课全，入 `昴星`。
- 三课备且无遥无克，入 `别责`。
- 两课备且无贼克，入 `八专`。
- 三者互斥，不再用“先尝试别责或八专，失败再回退”的补丁式结构。

## 风险

1. `涉害` 条文实现最复杂，若仅参考九宗门摘要而不引入更多细则，仍可能与目标站点少量偏差。
2. 返吟“有克”路径与常规贼克路径交叉较深，需要避免误改非返吟局。
3. 现有参考测试依赖旧算法输出，重构后应以条文和目标站点对照结果为准，必要时更新基线。
