# 大六壬专业计算页增加四柱与三传坐支设计

**目标**

1. 在大六壬专业计算页表格中增加 `年柱`、`月柱`、`日柱` 三列。
2. 为 `初传`、`中传`、`末传` 分别增加对应的 `坐支` 列。
3. 保持页面表格与导出 XLSX 字段完全一致，避免展示与导出内容不一致。

**现状**

- 大六壬“更多”页面实际对应 `src/components/DaLiuRenProPage.jsx`。
- 当前页面按日期范围批量计算后，只展示 `日期 / 初传 / 中传 / 末传`。
- 导出 XLSX 也仅导出这 4 列。
- `getDaLiuRenPaiPan` 已返回 `ganZhi.year/month/day` 以及 `tianPan`、`sanChuan`，具备生成目标列所需的数据。

**方案**

## 表格与导出口径

- 页面表格与 XLSX 统一输出以下列：
  - `日期`
  - `年柱`
  - `月柱`
  - `日柱`
  - `初传`
  - `初传坐支`
  - `中传`
  - `中传坐支`
  - `末传`
  - `末传坐支`
- 不采用“`初传=寅（坐申）`”这种合并字段形式，避免影响筛选、排序与后续数据处理。

## 三传坐支计算

- 三传本身是天盘地支。
- “坐支”按当前天地盘反查，即在 `tianPan` 中找到该三传地支所在位置，再映射回对应地盘支位。
- 该逻辑只在专业计算页本地使用，避免为单一页面需求扩大 `daliuren.js` 的对外 API。

## 错误处理与日志

- 若 `tianPan` 无效或无法反查出坐支，返回 `-`，并打印带 `[DaLiuRen][Pro]` 前缀的警告日志。
- 若整天批量计算失败，则新增列统一填 `-`，并保持现有失败标记与错误日志逻辑不变。

**影响范围**

- 修改 `src/components/DaLiuRenProPage.jsx`：
  - 批量结果行结构
  - 坐支反查辅助函数
  - 表格列定义
  - XLSX 导出字段
- 新增或更新一个大六壬专业页回归测试脚本，验证行数据包含新字段。

**验证**

- 页面计算后，表格能显示 `年柱 / 月柱 / 日柱` 与三传各自 `坐支`。
- 导出 XLSX 后，列顺序与页面一致。
- 运行大六壬相关回归测试，确认普通排盘逻辑未受影响。
